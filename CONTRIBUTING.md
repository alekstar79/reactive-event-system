# Reactive Event System (RES) Library

## Contributing

Thank you for your interest in contributing to EventBus! We appreciate your help in making this project better.

### Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/alekstar79/reactive-event-system.git`
3. Create a feature branch: `git checkout -b feature/amazing-feature`
4. Install dependencies: `npm install`

### Development

```bash
# Type checking
npm run type-check

# Run tests
npm test

# Run tests in watch mode
npm test -- --watch

# Run tests with UI
npm run test:ui

# Generate coverage report
npm run test:coverage

# Build the library
npm run build

# Build type definitions
npm run build:types
```

### Code Style

- We follow TypeScript best practices
- Use ES2024+ features
- Write clear, self-documenting code
- Add comments for complex logic
- Follow the existing code style

### Testing

All contributions must include tests:

```bash
npm test
```

We aim for >90% code coverage.

### Commit Messages

Use clear and descriptive commit messages:

```
feat: Add new feature description
fix: Fix bug description
docs: Update documentation
refactor: Refactor code section
test: Add or update tests
chore: Update dependencies
```

### Pull Requests

1. Ensure all tests pass: `npm test`
2. Ensure type checking passes: `npm run type-check`
3. Update documentation if needed
4. Provide a clear description of changes
5. Reference any related issues

### Issues

- Use clear, descriptive titles
- Provide reproduction steps for bugs
- Include relevant code snippets
- Mention your environment (Node.js version, OS, etc.)
